Clients place orders for event furniture ahead of their event; orders are managed by the OmniSpace admin team.This brief covers taking the PHP version of the Solar & Storage Live site live. You will decide on hosting. Once deployed, the same codebase will also be used for GITEX Kenya (May 2026).What's Already BuiltComplete PHP/MySQL ordering platform with branded catalogAdmin panel: orders, invoices (PDF), product management, image uploadAutomated emails: order confirmation to client + admin notificationTwo events pre-configured: Solar & Storage Live, and GITEX Kenya (May 19–21)All code is in the OmniShop PHP folder in Susan's Cowork folderTechnical RequirementsRequirementMinimum specPHP version7.4 or higher (PHP 8.x preferred)DatabaseMySQL 5.7+ or MariaDB 10.3+Web serverApache with mod_rewrite enabled (.htaccess support)SSL certificateRequired — HTTPS must work on the subdomainPHP extensionsPDO, PDO_MySQL, GD (for image processing), OpenSSLStorageAt least 500 MB for product images and invoicesSubdomainThe site should run at:https://shop.omnispace3d.comThis is already set in config.php. If a different subdomain is used, update SITE_URL in config.php before deployment.DNS: Create an A record (or CNAME) pointing shop.omnispace3d.com to the hosting server IP.Deployment Steps1. Set Up Hosting and DatabaseCreate a MySQL database and note down: host, database name, username, passwordEnsure PHP 7.4+ and Apache mod_rewrite are availableSet up the subdomain shop.omnispace3d.com pointing to the document root2. Configure config.phpUpdate these values in config.php before uploading:define('DB_HOST',   'your_db_host');define('DB_NAME',   'your_db_name');define('DB_USER',   'your_db_user');define('DB_PASS',   'your_db_password');define('SITE_URL',  'https://shop.omnispace3d.com');3. Upload All PHP FilesUpload the entire OmniShop PHP folder contents to the document root of shop.omnispace3d.comPreserve all subfolders: includes/, pages/, api/, lib/, static/, storage/The .htaccess file must be uploaded (it may be hidden — enable Show Hidden Files in your FTP client)✓  .htaccess is critical — without it all URLs except the homepage will return 404.4. Create Storage FoldersThese folders must exist and be writable by PHP:storage/invoices/static/images/products/In cPanel File Manager: create the folders, then set permissions to 755.5. Run SetupVisit in a browser:https://shop.omnispace3d.com/setup.phpThis creates all database tables and the default admin account.⚠  Delete or rename setup.php after running it — it should not remain publicly accessible.6. Add Admin Users (SQL)Run the SQL in ADD_ERIC_SQL.txt (in the OmniShop PHP folder) in phpMyAdmin to create Eric's admin account.7. Configure Email (Gmail App Password)Log in to Google with orders@omnispace3d.comGo to: myaccount.google.com/security → 2-Step Verification (enable if not on)Go to: myaccount.google.com/apppasswordsSelect Mail + Windows Computer → GenerateCopy the 16-character passwordIn OmniShop: Admin → Settings → paste into smtp_password → Save8. Upload Product ImagesSee the separate image brief (shared with Kevin). Images from the Python version should be compressed and batch-uploaded via Admin → Images → Batch Upload.Post-Launch ChecklistTestPass / NotesSite loads at https://shop.omnispace3d.comSSL padlock shows in browserAdmin login works (admin@omnispace3d.com)Catalog displays products with imagesTest order submitted — confirmation email receivedAdmin receives notification email for test orderInvoice PDF downloads correctlyAdmin → Products: can add/edit a productAdmin → Images: can upload a product imageAdmin → Users: Eric visible and able to log inKey Logins & ContactsItemDetailsAdmin panel URLhttps://shop.omnispace3d.com/adminDefault adminadmin@omnispace3d.com  /  OmniAdmin2026!Eric's admin accounterickiminya@gmail.com  —  see ADD_ERIC_SQL.txtOrders emailorders@omnispace3d.com  (receives all notifications)Susansusan.mboya@geminvest.co.keFiles ReferenceFile / FolderPurposeconfig.phpDatabase credentials and site URL — update before uploadsetup.phpRun once to create DB tables — delete after useADD_ERIC_SQL.txtSQL to run in phpMyAdmin to create Eric's accountUPLOAD_CHECKLIST.txtFull list of files to upload with priority orderincludes/events.phpEvent names, dates, and deadlinespages/admin/All admin panel pagesstatic/images/products/Product images (upload after launch)storage/invoices/Generated PDF invoices (auto-created)
